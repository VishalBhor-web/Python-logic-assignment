3.A primary key is a special column (or a set of columns) in a database table that uniquely identifies each row (record)
in that table. Essentially, it's used to make sure every record can be distinguished from every other record.

Why Must Primary Keys Be Unique and Non-Null?
Unique:
The primary key value must be unique for each row because it acts as the record's unique identifier. If two rows had the same primary 
key, there would be no way to tell them apart. For example, if two users had the same user ID, the system wouldn’t know which user is which.

Non-Null:
The primary key cannot be NULL because NULL means "unknown" or "missing." If a primary key was NULL, it wouldn’t be a valid identifier 
since NULL does not represent any concrete value, making it impossible to reliably find or reference that record.

How Primary Keys Help Identify Records
When you query the database, you often want to select, update, or delete specific records. The primary key allows you to precisely
locate these records without ambiguity.
Primary keys are used to establish relationships between tables. For example, a foreign key in another table will reference the 
primary key, linking related data together.
