A database schema is the formal blueprint that defines how data is organized, stored, and related within a database.

What a Schema Defines:
Table names: The specific entities (e.g., Users, Orders).
Columns: The attributes and data types (e.g., integer, text) for each table.
Constraints: Rules governing data integrity, including:
Primary keys: Unique row identifiers.
Unique/Non-null: Prevents duplicates or empty values.
Foreign keys: Establishes relationships between tables.

Why Schemas Are Important:
Data Consistency: Enforces uniform formats and rules across the dataset.
Error Prevention: Uses constraints to block invalid or missing data.
Scalability: Simplifies maintenance and long-term structural growth.
Collaboration: Provides a shared map for developers to work efficiently.
Performance: Optimizes storage and improves query speeds.
