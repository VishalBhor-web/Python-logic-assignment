2.The relational database mental model involves thinking of data as organized in tables, where:

A table represents a single entity type (e.g., User, Product).
A row is one record or instance of that entity.
A column defines an attribute or property of the entity (e.g., name, age).
Each table should represent only one entity to keep data organized, avoid confusion, and ensure the database 
design is clear and efficient. This simplifies querying, maintains consistency, and supports relationships between entities.
