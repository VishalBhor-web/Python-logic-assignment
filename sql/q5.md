5.To link data across tables in a relational database, we use specific identifiers called Keys.

Key Components:
Primary Key (PK): A unique ID for every row in a table (e.g., user_id in a Users table).
Foreign Key (FK): A column that references a PK in another table, creating a formal link (e.g., user_id inside an Orders table).
How it Works:
By placing a User's PK into the Orders table as an FK, you connect specific orders to the correct customer without 
duplicating their personal info.
